## [PrototypeAssembler](https://github.com/SpyDev14/PrototypeAssembler)
Приложение, которое подразумевалось как утилита для ивентологов и разработчиков ивентовых прототипов для SS14, которое позволяло бы им писать ивентовые прототипы по разным файлам, объединять их с помощью этой программы в один большой прототип и после грузить его в игру.

Сейчас все разработчики пишут всё в один файл, и если прототипов много - то работают с файлом по 500, 1000 или по 2000 строк YAML кода, что бывает неудобно.

Это мой первый серьёзный проект, там я работал с интерфейсами и абстракциями (некоторые говорят, что я излишне всё усложнил). Познакомился с DI.

Приложение по своей сути просто рекурсивно собирает и объединяет текстовые файлы с расширением .yaml / .yml в один большой, с добавлением некоторой информации о нём.

Было реализовано 2 подпроекта - Core и UI Shell.
Core - отвечает за сборку, UI Shell - за удобный UI. Core не зависит от реализации UI. Также можно работать с самим Core напрямую, запуская его с параметрами из консоли.
